Quality Gate¶

techrevati.runtime.quality_gate ¶

Quality Gate — Graduated quality levels for pass/fail decisions.

Replaces binary pass/fail with a four-level classification. QualityLevel is an IntEnum so >= comparison works naturally. Callers define their own mapping from observed metrics (confidence scores, test counts, signal strength) to a QualityLevel; the runtime stays opinion-free about what those numbers mean.

QualityLevel ¶

Bases: IntEnum

Ordered quality levels. Higher is stricter.
